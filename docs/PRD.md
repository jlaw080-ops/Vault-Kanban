# PRD: Vault Kanban

> **Product Requirements Document** — 이 앱이 무엇을 해결하는지 정의한다.
> 변경이 생기면 이 문서를 먼저 수정한 뒤 다른 문서를 맞춘다.

---

## 목표

로컬 Obsidian vault의 마크다운 노트를 **칸반 보드로 시각화·편집·AI 분석**하는 Electron 데스크톱 앱을 만든다. 노트는 오직 로컬 `.md` 파일에만 저장되며, Obsidian과 완전히 양방향 동기화된다.

**한 줄 요약**: Obsidian vault 위에 얹는 "제대로 된 칸반".

---

## 해결하려는 문제

1. Obsidian 노트가 많아질수록 "무엇을 어디까지 진행했는가"를 한눈에 보기 어렵다.
2. 여러 프로젝트가 섞여 있을 때 같은 프로젝트 노트들을 묶어서 보기 어렵다.
3. 외부 칸반 앱(Trello, Notion 등)을 쓰면 데이터가 외부에 종속되고 Obsidian과 이중 관리가 된다.

---

## 사용자

- **1인 사용자** (팀 공유 기능은 범위 밖)
- 건축설비/신재생에너지 VE/ZEB/Passive House 프로젝트를 Obsidian으로 관리하는 엔지니어
- macOS / Windows 모두 지원

---

## 핵심 기능

1. **Vault 연결 및 실시간 동기화** — 사용자가 선택한 폴더의 모든 `.md`를 파싱하고, Obsidian에서 수정하면 앱에도 즉시 반영된다 (chokidar 파일 감시).
2. **칸반 보드 (정석 기능)** — WIP Limit, 컬럼 정책(Definition of Done), 체류시간 색상 경고, 4가지 그룹핑(status/태그/폴더/AI 프로젝트)을 갖춘다.
3. **드래그앤드롭 이동** — 카드 이동 시 해당 `.md` 파일의 frontmatter를 직접 수정한다. `started`/`completed` 타임스탬프를 자동 기록한다.
4. **노트 에디터** — frontmatter 폼 UI + 마크다운 에디터를 우측 패널에 제공한다. Cmd+S 저장, 자동저장 지원.
5. **지표 대시보드** — 리드타임, 사이클타임, 처리량, 누적 흐름도(CFD)를 차트로 표시한다.
6. **AI 그룹핑** — Anthropic Claude API로 (A) 전체 노트 자동 프로젝트 분류, (B) 관련 노트 찾기 두 모드 제공.
7. **status 마이그레이션 도구** — 기존 자유형 상태값을 표준 5단계(백로그/예정/진행중/검토/완료)로 일괄 변환. 자동 백업 + 롤백 지원.

---

## 비기능 요구사항

- **로컬 우선**: 노트 데이터는 사용자 컴퓨터에만 존재한다. AI 분석 시에만 선택적으로 외부 API 호출.
- **성능**: vault에 노트 100개가 있어도 최초 로드 1초 이내.
- **보안**: Anthropic API 키는 `safeStorage`로 OS 키체인에 암호화 저장. 평문 저장 금지.
- **데이터 무결성**: 마이그레이션 같은 파괴적 작업은 반드시 자동 백업 선행.

## MVP 제외 사항 (= 이번 버전에서 만들지 않는다)

> 이 목록이 핵심이다. AI가 scope를 늘리려 할 때 여기를 근거로 거절한다.

- **팀 공유 / 실시간 협업** — 1인용만 지원

- **스윔레인(가로 분할)** — v0.2 이후로 미룸
- **Obsidian Canvas 연동** — v0.2 이후
- **회고 노트 자동 생성** — v0.2 이후
- **코드 서명 인증서** — 개인용이므로 OS 경고 감수
- **Tauri 등 대체 프레임워크 검토** — Electron으로 고정
- **자체 DB(SQLite 등) 도입** — `.md` 파일이 곧 저장소

## 성공 기준

다음이 모두 충족되면 MVP 완성:

1. 실제 내 Obsidian vault를 연결해서 칸반 보드가 정상 작동
2. 카드를 드래그하면 `.md` 파일 frontmatter의 `status`가 실제로 변경됨
3. Obsidian에서 노트를 수정하면 1초 내 앱 UI에 반영됨
4. AI 그룹핑 실행 시 의미 있는 프로젝트 분류가 나옴
5. macOS `.dmg` / Windows `.exe` 빌드가 GitHub Actions로 자동 생성됨
6. 실제 vault로 **1주일 이상 사용**하면서 치명적 버그 없음
