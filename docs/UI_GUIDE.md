# UI Guide: Vault Kanban

> **AI가 기본 스타일 그대로 출력하지 않게 하는 최후의 방어선**이다.
> 이 문서 없이 실행하면 AI는 전형적인 "AI 슬롭" 디자인을 내놓는다.

---

## 디자인 원칙

1. **기능이 먼저, 장식은 최소** — 칸반 보드는 매일 본다. 애니메이션·그라데이션은 피로를 준다.
2. **Obsidian과 시각적 조화** — vault 파일을 조작하는 앱이므로 Obsidian 기본 UI와 톤이 어긋나지 않아야 한다.
3. **정보 밀도 > 여백 과시** — 한 화면에 100장 카드가 떠도 읽히게 한다. SaaS 랜딩 페이지처럼 여백을 낭비하지 않는다.
4. **색은 시맨틱으로만** — 색 = 의미(상태, 경고, 우선순위). 장식용 색 금지.
5. **접근성 필수** — 색맹 테스트 통과. 색 + 아이콘/텍스트 병행.

---

## AI 슬롭 안티패턴 — 절대 하지 마라

| 금지 사항 | 이유 |
|-----------|------|
| `backdrop-filter: blur()` (glass morphism) | AI 템플릿의 가장 흔한 징후 |
| `bg-gradient-to-r from-purple-500 to-pink-500` | 보라→분홍 그라데이션은 AI 랜딩의 클리셰 1위 |
| 텍스트에 gradient (`bg-clip-text`) | AI가 만든 SaaS 페이지의 표식 |
| `box-shadow` 글로우 애니메이션 / 네온 발광 | 네온 글로우 = AI 슬롭 |
| 보라/인디고를 브랜드 색으로 | "AI = 보라색" 클리셰 |
| 모든 카드에 `rounded-2xl` / `rounded-3xl` | 균일한 대형 둥근 모서리 = 템플릿 느낌 |
| 배경에 떠 있는 gradient orb (blur된 원) | 모든 AI 랜딩 페이지의 장식 |
| Hero 섹션, Feature grid 3x3 같은 랜딩 페이지 레이아웃 | 이 앱은 작업 도구다. 랜딩 페이지가 아니다 |
| `animate-pulse`, `animate-bounce`를 장식용으로 | 로딩 표시 외 금지 |
| 이모지를 본문에 남용 (🚀✨🎉) | UI 텍스트에는 **기능 의미 있는 아이콘만** (lucide-react) |
| "Powered by AI" 같은 문구 | 부가 기능일 뿐이다. 자랑하지 않는다 |

## 색상

**shadcn/ui slate 팔레트를 기본으로 사용한다.** 추가 색은 아래 시맨틱 용도로만.

### 시맨틱 색상

| 용도 | Light | Dark | 비고 |
|------|-------|------|------|
| 배경 | `slate-50` / `white` | `slate-950` | |
| 카드 배경 | `white` | `slate-900` | |
| 주 텍스트 | `slate-900` | `slate-100` | |
| 보조 텍스트 | `slate-500` | `slate-400` | |
| 테두리 (기본) | `slate-200` | `slate-800` | |
| 성공 | `emerald-600` | `emerald-500` | 완료 상태 |
| 경고 | `amber-500` | `amber-400` | 체류시간 3~6일 |
| 위험 | `red-500` | `red-500` | 체류시간 7일+, WIP 초과 |

### 상태별 컬럼 색 바 (컬럼 헤더 위 3px)

| 상태 | 색 |
|------|---|
| 백로그 | `slate-400` |
| 예정 | `blue-500` |
| 진행중 | `amber-500` |
| 검토 | `violet-500` (여기만 예외적으로 violet 허용 — 상태 구분 기능 목적) |
| 완료 | `emerald-500` |

> **예외 주의**: 위 violet은 **시맨틱 용도**(상태 구분)로만. 장식/브랜딩 용도로 violet 쓰지 말 것.

### 우선순위 점 색상

| 우선순위 | 색 |
|---------|---|
| high | `#ef4444` (red-500) |
| mid | `#f59e0b` (amber-500) |
| low | `#6b7280` (gray-500) |

## 타이포그래피

- **폰트**: 시스템 기본.
  ```css
  font-family: -apple-system, "Segoe UI", "Malgun Gothic", sans-serif;
  ```
- **제목**: `font-semibold` (700 금지, 과하게 굵음).
- **본문**: `font-normal text-sm` (14px 기준).
- **보조 텍스트**: `text-xs text-slate-500`.
- **한글 자간 조정 금지**. 시스템 기본을 유지.

## 컴포넌트 스타일

### 카드 (`KanbanCard`)

- `rounded-md` (8px). **절대 `rounded-xl` 이상 금지.**
- `border border-slate-200 dark:border-slate-800` 기본.
- padding: `p-3`.
- hover: `hover:border-slate-300 dark:hover:border-slate-700` (테두리만 변화. shadow 금지).
- 체류시간 경고는 테두리 색 변화:
  - 3일 미만: 기본
  - 3~6일: `border-amber-400`
  - 7일 이상: `border-red-500`

### 컬럼 (`KanbanColumn`)

- `bg-slate-50 dark:bg-slate-900`.
- `rounded-md`, `p-3`.
- 헤더에 상태별 3px 색 바 (위 시맨틱 표 참조).
- 카드 간 간격: `space-y-2` (8px).

### 버튼

- shadcn/ui `Button` 기본 variant 사용.
- **Primary 버튼에 gradient 절대 금지.** `bg-slate-900 text-white` 같은 단색.

### 입력 필드

- shadcn/ui `Input` 기본.
- 포커스 시 링: `ring-1 ring-slate-400`. **glow 금지.**

## 레이아웃

```
┌─────────────────────────────────────────────────────────────┐
│  TopBar  (높이 48px)                               [설정]   │
├────────┬────────────────────────────────────────┬───────────┤
│        │  컨트롤 바 (그룹핑/정렬/필터/AI)       │           │
│Sidebar │  ─────────────────────────────         │  Editor   │
│        │  ┌─────┬─────┬─────┬─────┬─────┐       │           │
│  너비  │  │백로그│예정 │진행중│검토 │완료 │       │  너비 40% │
│  200px │  │     │     │(2/3)│     │     │       │  리사이저 │
│        │  │카드 │카드 │카드 │카드 │카드 │       │  가능     │
│        │  │카드 │     │카드 │     │     │       │           │
└────────┴────────────────────────────────────────┴───────────┘
```

### 여백 규칙

- 전체 패딩: `p-4`.
- 카드 간 간격: `gap-2` (8px).
- 컬럼 간 간격: `gap-3` (12px).
- 섹션 간 간격: `space-y-6`.

### 반응형

- 최소 창 크기: **1024 × 720**. 그 이하는 지원 안 함.
- 창 너비 1280px 미만: 사이드바 아이콘만 표시 (텍스트 숨김).
- 에디터가 열렸을 때 칸반 영역이 600px 이하가 되면 에디터 전체화면 모드로 전환.

## 애니메이션

**허용되는 애니메이션** (이 목록 외 모두 금지):

| 위치 | 효과 | 지속 시간 |
|------|------|-----------|
| 드래그 중인 카드 | 살짝 들림 (`scale-105`, `shadow-lg`) | 즉시 |
| 드롭 시 카드 | transform 전이 | 150ms ease-out |
| 모달 열림/닫힘 | fade + scale (shadcn/ui 기본) | 200ms |
| 토스트 슬라이드 인 | translate-y | 200ms |
| 로딩 스피너 | lucide-react `Loader2 className="animate-spin"` | - |

**금지되는 애니메이션**:
- Hover 시 카드 scale
- 아이콘 회전/펄스 (로딩 외)
- 배경 gradient shift
- 페이드 인 on scroll

## 접근성

- **색상만으로 정보 전달 금지**. 우선순위/체류시간은 색 + 아이콘/텍스트 병행.
  - 예: high 우선순위는 빨간 점 + "high" 텍스트.
- 모든 인터랙티브 요소에 `aria-label`.
- `@dnd-kit` 키보드 센서 활성화 → Tab + Space + 방향키로 카드 이동 가능하게 한다.
- 포커스 링(`focus-visible:ring-2`) 제거 금지.

---

## 아이콘

- **라이브러리**: `lucide-react` 고정.
- **금지**: emoji를 UI 아이콘으로 사용 (PRD 텍스트 설명에는 OK).
- 크기: 대부분 `size-4` (16px). 카드 작은 메타 정보는 `size-3`.
- 색: 기본 `text-slate-600 dark:text-slate-400`.

---

## 다크/라이트 모드

- 시스템 설정 자동 감지 (`prefers-color-scheme`).
- 설정에서 수동 전환 가능 (`light` / `dark` / `system`).
- **Tailwind의 `dark:` 변형을 모든 색상 클래스에 반드시 병기**. 누락 시 다크 모드에서 깨짐.

## 체크리스트 (컴포넌트 구현 시 매번 확인)

- [ ] `rounded-2xl` 이상 쓰지 않았다
- [ ] `bg-gradient-*` 쓰지 않았다
- [ ] `backdrop-blur` 쓰지 않았다
- [ ] 보라/인디고를 브랜드 색으로 안 썼다 (검토 컬럼 예외)
- [ ] 모든 색상 클래스에 `dark:` 변형 병기했다
- [ ] `aria-label`이 있다
- [ ] 키보드 조작이 가능하다
- [ ] 최소 너비 1024px에서 레이아웃이 깨지지 않는다
